# Code of Conduct

NiceChunk contributors and maintainers are expected to keep technical discussion respectful, specific, and focused on improving the project.

## Expected Behavior

- Be direct about bugs, risks, and tradeoffs without personal attacks.
- Keep reviews grounded in code, reproducible behavior, documentation, or project scope.
- Respect security boundaries by not posting private credentials, server details, or exploit instructions in public threads.
- Keep discussions relevant to nicechunk-worldgen and the NiceChunk repository surface being changed.

## Unacceptable Behavior

- Harassment, threats, discrimination, or repeated hostile comments.
- Publishing private information, credentials, server details, or non-public operational material.
- Deliberately derailing issue or pull request discussions.

## Enforcement

Maintainers may edit, hide, lock, or remove comments and issues that violate these expectations. Severe or repeated violations may result in blocked participation.
